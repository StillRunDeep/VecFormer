## 10.1 注册表模式（Registry Pattern）

### 问题：如何让 launch.py 不依赖具体的模型类？

如果直接 import，每次换模型都要修改 `launch.py`：

```python
# 不好的写法：紧耦合
from model.vecformer import VecFormer, VecFormerTrainer

model = VecFormer(config)
```

### 解决方案：注册表

```python
# model/__init__.py

_registry: dict = {}  # 全局字典：名称 → 构建函数

def register_model(name: str):
    """装饰器工厂：把被装饰的函数注册到 _registry"""
    def decorator(build_fn):
        _registry[name] = build_fn
        return build_fn
    return decorator

def build_model(model_args_path: str):
    """根据配置文件里的名称，找到对应的构建函数并调用"""
    config = load_yaml(model_args_path)
    name   = config["model_name"]              # e.g. "vecformer"
    return _registry[name](config["model_args"])
```

```python
# model/vecformer/__init__.py

@register_model("vecformer")                   # 一行代码完成注册
def build(model_args: dict):
    config = VecFormerConfig(**model_args)
    model  = VecFormer(config)
    return model, VecFormerTrainer
```

```python
# launch.py（完全不需要 import VecFormer）

model, TrainerClass = build_model("configs/model/vecformer.yaml")
# 内部：_registry["vecformer"](model_args)
```

### 添加新模型的流程

```
1. 新建文件 model/my_new_model/__init__.py
2. 写：
   @register_model("my_model")
   def build(model_args):
       return MyModel(...), MyTrainer
3. 在 model/__init__.py 里加一行：
   import model.my_new_model
4. 改配置：configs/model/my_model.yaml 里 model_name: my_model
5. 完成！launch.py 无需修改
```

数据集同理，使用 `@register_dataset("floorplancad")`（`data/__init__.py`）。

## 10.2 HuggingFace Trainer 模式

### 为什么用 HuggingFace？

从零实现一个生产级训练器需要处理：
- 多 GPU 分布式训练
- 混合精度（fp16/bf16）
- 梯度累积
- 学习率调度
- 检查点保存/恢复
- TensorBoard 日志
- 进度条显示
- ……

HuggingFace `Trainer` 已经实现了所有这些，只需继承并重写需要定制的部分。

### 继承关系

```
HuggingFace Trainer（基类，约 5000 行代码）
        │
        └── VecFormerTrainer（重写 3 个方法）
```

### VecFormerTrainer 重写了什么

**代码位置**：`model/vecformer/vecformer_trainer.py`

```python
class VecFormerTrainer(Trainer):

    def compute_loss(self, model, inputs, return_outputs=False, **kwargs):
        """重写：告诉 Trainer 如何计算损失"""
        outputs = model(**inputs)
        loss = outputs.loss

        if return_outputs:
            return loss, outputs
        return loss

    def prediction_step(self, model, inputs, prediction_loss_only, ...):
        """重写：告诉 Trainer 推理时如何收集输出（我们要 metric_states）"""
        with torch.no_grad():
            outputs = model(**inputs)

        # 返回 (loss=None, logits=metric_states, labels=None)
        # Trainer 会把所有 batch 的 logits 汇总
        return None, outputs.metric_states, None

    def compute_metrics(self, eval_pred):
        """重写：告诉 Trainer 如何从汇总的 metric_states 计算最终指标"""
        all_metric_states = eval_pred.predictions
        return evaluator.compute_final_metrics(all_metric_states)
        # 返回 {"PQ": 0.512, "SQ": 0.734, "RQ": 0.698, ...}
```

### Trainer 自动处理的事情

```
trainer.train() 调用时，Trainer 自动：
  ✅ 把模型分配到多 GPU（DDP）
  ✅ 包装 DataLoader（多进程加载，pin_memory）
  ✅ 每步调用 compute_loss() 获取 loss
  ✅ loss.backward() 反向传播
  ✅ 梯度裁剪
  ✅ optimizer.step()
  ✅ lr_scheduler.step()
  ✅ 每 eval_steps 步调用 prediction_step() 和 compute_metrics()
  ✅ 按 metric_for_best_model 决定是否保存检查点
  ✅ 写 TensorBoard 日志
  ✅ 打印训练进度
```

## 10.3 PointModule / PointSequential 模式

### 问题：为什么不用普通的 nn.Module？

普通 PyTorch 的模块调用方式：

```python
# 标准写法
x = layer1(x)
x = layer2(x)
x = layer3(x)
```

但 PointTransformerV3 中，每一层处理的数据不只是一个张量，而是一个包含多个属性的"点云对象"：

```python
# point 是一个 addict.Dict，包含：
point.feat        # (N, C)  特征
point.coord       # (N, 3)  坐标
point.offset      # (B,)    batch 边界
point.grid_coord  # (N, 3)  体素网格坐标
point.serialized_order    # 序列化顺序（4种）
point.serialized_inverse  # 反序列化索引
# ... 以及各种缓存字段
```

如果用 `nn.Sequential`，这些额外信息没法传递——`nn.Sequential` 只能传递单一张量。

### 解决方案：PointModule

**代码位置**：`point_transformer_v3/model.py:PointModule / PointSequential`

```python
class PointModule(nn.Module):
    """基类：前向传播接收和返回 point 字典"""
    def forward(self, point: addict.Dict) -> addict.Dict:
        raise NotImplementedError

class PointSequential(PointModule):
    """像 nn.Sequential，但传递 point 字典而非张量"""
    def forward(self, point):
        for module in self._modules.values():
            if isinstance(module, PointModule):
                point = module(point)   # 传 point 字典
            else:
                point.feat = module(point.feat)  # 普通层只处理 feat
        return point
```

使用方式：

```python
# 替代 nn.Sequential
self.enc_layer1 = PointSequential(
    Block(channels=32, enable_flash=True),  # PointModule
    Block(channels=32, enable_flash=True),  # PointModule
    nn.LayerNorm(32),                        # 普通 nn.Module，自动只处理 feat
)

# 调用时：
point = self.enc_layer1(point)  # 输入输出都是 point 字典
```

## 10.4 PretrainedConfig / PreTrainedModel 模式

### 为什么继承 HuggingFace 的基类？

```python
class VecFormerConfig(PretrainedConfig):
    ...

class VecFormer(PreTrainedModel):
    config_class = VecFormerConfig
    ...
```

好处：

**1. 自动序列化配置**

```python
# 保存（自动生成 config.json）
config.save_pretrained("outputs/checkpoint/")

# 加载（从 config.json 恢复）
config = VecFormerConfig.from_pretrained("outputs/checkpoint/")
model  = VecFormer(config)
```

**2. 自动权重保存/加载**

```python
# Trainer 使用 model.save_pretrained() 保存
# 加载时：
model = VecFormer.from_pretrained("outputs/checkpoint/")
# 自动加载权重 + 配置
```

**3. 兼容 HuggingFace 生态**

`push_to_hub()`、`from_pretrained("用户名/模型名")` 等功能开箱即用。

## 10.5 Monkey Patch（猴子补丁）

**代码位置**：`utils/patches/`

在不修改第三方库源码的情况下，在运行时替换其部分函数。

```python
# utils/patches/logging_patch.py（示意）

import transformers.trainer

# 保存原始函数
_original_log = transformers.trainer.Trainer.log

def _patched_log(self, logs, **kwargs):
    # 添加自定义逻辑（如改变日志格式）
    logs = format_logs(logs)
    return _original_log(self, logs, **kwargs)

# 运行时替换
transformers.trainer.Trainer.log = _patched_log
```

本项目用猴子补丁来定制：
- 日志格式（`logging_patch.py`）
- 进度条显示（`printer_callback_patch.py`）
- 训练参数扩展（`training_arguments_patch.py`，添加 `launch_mode` 等自定义参数）

```python
# launch.py 一开始就调用
def main():
    apply_patches()   # 在 import HuggingFace 之后，使用之前，打上补丁
    ...
```

## 10.6 设计模式总结

| 模式 | 解决的问题 | 好处 |
|------|---------|------|
| **注册表** | 解耦模型/数据集代码与 launch.py | 新增模型不需要修改入口文件 |
| **HuggingFace Trainer** | 避免重复实现训练循环 | 分布式、混合精度、检查点等自动处理 |
| **PointModule** | 传递多属性点云对象 | 所有层可以访问坐标、offset 等元数据 |
| **PretrainedConfig** | 统一配置保存/加载 | 与 HuggingFace 生态完全兼容 |
| **Monkey Patch** | 定制第三方库行为 | 不 fork 库，升级库时改动最小 |
