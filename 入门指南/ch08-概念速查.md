## 数据相关

| 术语 | 含义 | 代码位置 |
|------|------|---------|
| `primitive` / `prim` | **图元**：一条线段（CAD 的基本元素）| `dataclass_define.py` |
| `primitive_id` | 图元类型 ID，同一个建筑符号（如一扇门）的所有线段共享相同的 primitive_id | `dataclass_define.py` |
| `layer_id` | CAD 图层 ID，建筑师用图层组织不同类型的线段 | `dataclass_define.py` |
| `semantic_id` | 语义类别标签（0=门，1=窗，...，34=背景） | `dataclass_define.py` |
| `instance_id` | 实例 ID，同一图内区分同类的不同个体 | `dataclass_define.py` |
| `SVGData` | 原始数据结构，Python 列表，从 JSON 加载 | `dataclass_define.py` |
| `VecData` | 处理后的数据结构，PyTorch 张量，送入模型 | `dataclass_define.py` |
| `cu_seqlens` | **累计序列长度**（Cumulative Sequence Lengths）。例：`[0, 300, 800, 1000]` 表示第0张图有300条线，第1张有500条，第2张有200条。取第 i 张图：`tensor[cu_seqlens[i]:cu_seqlens[i+1]]` | 几乎所有 forward 函数 |
| `cu_numprims` | 累计图元数，功能同 cu_seqlens，用于标签处理 | `modeling_vecformer.py` |
| `viewBox` | SVG 图纸的边界框 `[minx, miny, width, height]`，用于坐标归一化 | `floorplancad.py` |
| `prim_lengths` | 每条线段的欧氏长度，用于加权评估（长线段的错误影响更大）| `dataclass_define.py` |

## 模型结构相关

| 术语 | 含义 | 代码位置 |
|------|------|---------|
| `PTv3` / `PointTransformerV3` | 骨干特征提取网络，基于 PointTransformer V3 改编，专门处理点云/线段数据 | `point_transformer_v3/model.py` |
| `SerializedAttention` | PTv3 的核心注意力模块，先对线段排序（序列化），再做局部窗口注意力 | `point_transformer_v3/model.py:329` |
| `patch_size` | 序列化注意力的窗口大小，默认 1024（每次对 1024 条线做注意力）| `configuration_vecformer.py:28` |
| `stride` | 编码器下采样倍率，默认 `(2,2,2,2)`（每层线段数减半）| `configuration_vecformer.py:24` |
| `enc_depths` | 各编码器层的 Transformer Block 数量 `(2,2,2,6,2)`（第4层最深）| `configuration_vecformer.py:25` |
| `enc_channels` | 各编码器层的特征维度 `(32,64,128,256,512)` | `configuration_vecformer.py:26` |
| `dec_channels` | 各解码器层的特征维度 `(64,64,128,256)` | `configuration_vecformer.py:30` |
| `CPE` | Conditional Positional Encoding（条件位置编码），用稀疏卷积提取邻域位置信息 | `point_transformer_v3/model.py:Block` |
| `skip connection` | 跳跃连接，编码器层的特征在解码器对应层拼接回来，保留低级细节 | `point_transformer_v3/model.py:Upsample` |
| `GridPool` | PTv3 的下采样操作，按体素格子（Grid）选择代表点 | `point_transformer_v3/model.py` |
| `VecBackbone` | 向量骨干网络，包含编码器+解码器，处理线段特征（注：是对 PTv3 的进一步封装）| `vec_backbone/vec_backbone.py` |
| `CADDecoder` | CAD 解码器，通过查询向量机制预测实例和语义 | `cad_decoder/cad_decoder.py` |
| `query vector` / `queries` | **查询向量**，CAD 解码器中代表"候选实例"的可学习向量，通过注意力机制从特征中提取信息 | `modeling_vecformer.py`, `cad_decoder.py` |
| `FusionLayerFeatsModule` | 图层融合模块，把同一 CAD 图层的线段特征聚合，增强图层级别的上下文 | `modules/fusion_layer_feats_module.py` |
| `GroupFeatFusion` | 按 ID 分组聚合特征（scatter mean pooling），用于图层融合和图元融合 | `modules/group_feat_fusion.py` |

## 注意力机制相关

| 术语 | 含义 | 代码位置 |
|------|------|---------|
| `Q / K / V` | Query（查询）/ Key（键）/ Value（值），注意力机制的三要素 | `attention.py` |
| `qkv_packed` | Q、K、V 打包成一个张量 `(N, 3, H, D)`，形状紧凑，flash_attn 需要这种格式 | `attention.py:35` |
| `VarlenSelfAttention` | **变长自注意力**，同一张图的线段内部做自注意力 | `modules/attention.py:6` |
| `VarlenCrossAttention` | **变长交叉注意力**，查询向量从骨干特征中提取信息 | `modules/attention.py:145` |
| `VarlenSelfAttentionWithRoPE` | 带旋转位置编码的自注意力，位置信息通过旋转变换注入 Q/K | `modules/attention.py:53` |
| `RoPE` | Rotary Position Embedding（旋转位置编码），比绝对位置编码更好地保留相对位置信息 | `modules/attention.py:_apply_rope` |
| `flash_attn` | Flash Attention 库，内存高效的注意力实现，速度比标准注意力快 3–5 倍，需要 GPU sm_75+ | `attention.py:5` |
| `upcast_attention` | 在 fp32 下计算注意力分数（不启用 Flash Attention 时）。精度高但更慢 | `point_transformer_v3/model.py:342` |
| `upcast_softmax` | 在 fp32 下计算 softmax（不启用 Flash Attention 时）| `point_transformer_v3/model.py:343` |
| `enable_flash` | 是否启用 Flash Attention。RTX 3090+/A100 建议开启；2080 Ti 自动回退 | `configuration_vecformer.py:42` |
| `max_seqlen` | 当前 batch 中最长序列的长度，Flash Attention 需要此参数 | `attention.py:37` |
| `head_dim` | 每个注意力头的维度 = `embed_dim / n_heads` | `attention.py:21` |

## 训练相关

| 术语 | 含义 | 代码位置 |
|------|------|---------|
| `iter_pred` | 迭代预测，解码器的每一层都输出预测并计算损失，加速收敛 | `configuration_vecformer.py:63` |
| `query_thr` | 训练时采样查询向量的阈值，只有骨干特征置信度高于此阈值的位置才作为查询 | `configuration_vecformer.py:17` |
| `Hungarian matching` | 匈牙利算法，用于训练时将模型预测与真实标注做最优配对 | `criterion/instance_criterion.py` |
| `label_smoothing` | 标签平滑，把硬标签（0/1）软化为（ε/(C-1) / 1-ε），防止过拟合 | `configuration_vecformer.py:77` |
| `warmup` | 学习率预热，训练初期从小值线性增大到目标值，防止大梯度破坏初始化 | `configs/vecformer.yaml` |
| `weight_decay` | 权重衰减，L2 正则化的一种形式，防止参数过大 | `configs/vecformer.yaml` |
| `AdamW` | Adam 优化器 + 正确实现的权重衰减（Weight Decay）| `configs/vecformer.yaml` |
| `DropPath` | 随机路径丢弃，训练时随机跳过某些残差路径，防止过拟合 | `point_transformer_v3/model.py` |
| `torchrun` | PyTorch 多 GPU 分布式训练启动命令，替代旧版 `python -m torch.distributed.launch` | `scripts/train.sh` |
| `NCCL` | NVIDIA Collective Communications Library，GPU 间通信库，用于 all-reduce 梯度 | `launch.py` |
| `DDP` | DistributedDataParallel，数据并行：每张 GPU 独立前向/反向，梯度 all-reduce 同步 | HuggingFace Trainer 内部 |

## 评估相关

| 术语 | 含义 | 代码位置 |
|------|------|---------|
| `PQ` | Panoptic Quality（全景质量）= SQ × RQ，主要评估指标 | `evaluator/evaluator.py` |
| `SQ` | Segmentation Quality（分割质量）= 匹配对的平均 IoU | `evaluator/evaluator.py` |
| `RQ` | Recognition Quality（识别质量）= F1 分数 = TP/(TP+0.5FP+0.5FN) | `evaluator/evaluator.py` |
| `IoU` | Intersection over Union（交并比）= \|预测∩真实\|/\|预测∪真实\|，值域 [0,1] | `evaluator/evaluator.py` |
| `TP` | True Positive，正确检测到的实例（预测与真实匹配且 IoU > 阈值）| `evaluator/evaluator.py` |
| `FP` | False Positive，多余的预测（没有对应的真实实例，或 IoU 太低）| `evaluator/evaluator.py` |
| `FN` | False Negative，漏检的实例（真实中有，但模型未预测出）| `evaluator/evaluator.py` |
| `iou_threshold` | IoU 阈值，默认 0.5，超过此值才算 TP | `configuration_vecformer.py:100` |
| `pred_score_thr` | 置信度过滤阈值，低于此值的实例预测被丢弃 | `configuration_vecformer.py:94` |
| `mask_logit_thr` | 掩码二值化阈值，logit 大于此值的线段被认为属于该实例 | `configuration_vecformer.py:95` |
| `num_topk_preds` | 推理时保留的最大实例数（按置信度取 Top-K）| `configuration_vecformer.py:89` |
| `vector_nms` | 针对向量图形的非极大值抑制，去除重复的实例预测 | `modules/__init__.py` |

| `einops` | 张量重排操作（rearrange / repeat），比 reshape 更易读 | 辅助模块 |
| `addict` | 支持 dict.key 访问方式的字典，用于 point 对象 | `point_transformer_v3/model.py` |
