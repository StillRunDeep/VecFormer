## 9.1 推理 vs 训练的区别

| 维度 | 训练模式 | 推理模式 |
|------|---------|---------|
| 梯度计算 | ✅ 计算梯度 | ❌ `torch.no_grad()` 关闭 |
| 参数更新 | ✅ 更新参数 | ❌ 不更新 |
| 查询向量数 | 动态采样（Q≤N）| 全部线段（Q=N）|
| Dropout | ✅ 随机丢弃 | ❌ 关闭（`model.eval()`）|
| 输出 | loss 用于反向传播 | 具体预测结果 |

触发推理：`model.eval()` + 加载 checkpoint + 调用 `trainer.evaluate()`

## 9.2 推理的起点：CAD 解码器原始输出

CAD 解码器第 6 层（最后一层）输出原始预测，形状含义如下：

```python
# 假设：一张图，N=800 条线段，Q=800 个查询向量，35 个类别

pred_sem_labels   # (N, 35) = (800, 35)
                  # 每条线段属于每个类别的原始分数（logits，未经 softmax）
                  # 值域 (-∞, +∞)，负值=低概率，正值=高概率

pred_inst_labels  # (Q, 35) = (800, 35)
                  # 每个查询向量预测的实例类别 logits

pred_inst_scores  # (Q, 1)  = (800, 1)
                  # 每个查询向量的置信度 logits
                  # 经过 sigmoid 后变成 [0,1] 的概率

pred_inst_masks   # (Q, N)  = (800, 800)
                  # 查询向量 i 与线段 j 的相似度 logits
                  # 经过 sigmoid 后，值越大表示线段 j 越可能属于实例 i
```

---

## 9.3 第一步：语义预测（predict_semantic）

**代码位置**：`modeling_vecformer.py → predict_semantic()`

```python
def predict_semantic(pred_sem_labels):
    # pred_sem_labels: (N, 35)

    # Softmax：把 logits 变成概率分布（每行和为 1）
    probs = F.softmax(pred_sem_labels, dim=-1)   # (N, 35)

    # 取最大概率对应的类别
    pred_classes = probs.argmax(dim=-1)    # (N,)  类别 ID
    pred_scores  = probs.max(dim=-1).values # (N,)  最大概率值（置信度）

    return pred_classes, pred_scores

# 输出示例（5条线段）：
# pred_classes = [3, 3, 7, 7, 12]   → [门, 门, 窗, 窗, 墙]
# pred_scores  = [0.92, 0.87, 0.78, 0.95, 0.63]
```

## 9.4 第二步：实例预测（predict_instance）

**代码位置**：`modeling_vecformer.py → predict_instance()`

这是推理中最复杂的步骤，包含多层过滤。

### Step 1：置信度过滤

```python
# 对每个查询向量计算置信度
scores = torch.sigmoid(pred_inst_scores).squeeze(-1)  # (Q,)  值域 [0,1]

# 只保留置信度 > pred_score_thr=0.5 的查询向量
keep_mask = scores > 0.5                   # (Q,)  布尔掩码
scores = scores[keep_mask]                 # (K,)  K = 保留的查询数
pred_inst_labels = pred_inst_labels[keep_mask]  # (K, 35)
pred_inst_masks  = pred_inst_masks[keep_mask]   # (K, N)
```

### Step 2：计算实例类别

```python
# 对保留的查询向量，取最大类别概率
inst_probs   = F.softmax(pred_inst_labels, dim=-1)  # (K, 35)
inst_classes = inst_probs.argmax(dim=-1)             # (K,)  实例类别
inst_cls_scores = inst_probs.max(dim=-1).values      # (K,)  类别置信度
```

### Step 3：掩码激活

```python
# 把掩码 logits 转成概率
masks = torch.sigmoid(pred_inst_masks)  # (K, N)  值域 [0,1]

# 二值化：概率 > mask_logit_thr=0.3 的线段属于这个实例
binary_masks = (masks > 0.3)            # (K, N)  布尔矩阵
```

### Step 4：过滤空实例

```python
# 去掉没有线段的预测（避免幽灵实例）
mask_counts = binary_masks.sum(dim=-1)  # (K,)  每个实例的线段数
valid = mask_counts > 0                 # 只保留非空实例
```

### Step 5：对象归一化（Object Normalization）

```python
# 如果 use_obj_normalization=True（默认开启）
# 用实例内线段的长度对置信度进行归一化
# 防止：一条很长的线段被错误地当作"整个实例"

prim_lengths = ...   # (N,)  每条线段的长度
for i in range(K):
    # 实例 i 内所有线段的总长度
    total_length = prim_lengths[binary_masks[i]].sum()
    # 用总长度加权分数（更长的实例，分数应该更稳定）
    scores[i] = scores[i] * normalize_factor(total_length)
```

### Step 6：取 Top-K

```python
# 最多保留 num_topk_preds=600 个实例
topk_scores, topk_indices = scores.topk(min(600, len(scores)))
final_masks   = binary_masks[topk_indices]   # (Top-K, N)
final_classes = inst_classes[topk_indices]   # (Top-K,)
final_scores  = topk_scores                  # (Top-K,)
```

## 9.5 第三步：全景融合（predict_panoptic）

**代码位置**：`modeling_vecformer.py → predict_panoptic()`

将语义预测和实例预测合并，给每条线段分配最终的 `(类别, 实例ID)` 对。

### 处理 thing 类（有实例的类别，0–29）

```python
# 对每个实例预测，按置信度从高到低处理：
panoptic_seg = torch.zeros(N, dtype=torch.long)  # (N,) 初始为 0（未分配）

for inst_score, inst_class, inst_mask in zip(sorted_by_score):
    # 只有 thing 类才做实例分配
    if inst_class not in thing_class_idxs:
        continue

    # 找到属于这个实例的线段（尚未被更高置信度实例占用）
    available = (panoptic_seg == 0) & inst_mask   # 未被占用 且 在此实例掩码内

    # 如果有足够的可用线段（避免和高置信度实例抢同一片区域）
    if available.sum() / inst_mask.sum() > iou_threshold:
        panoptic_seg[available] = current_instance_id
        current_instance_id += 1
```

### 处理 stuff 类（背景类，30–34）

```python
# Stuff 类没有实例 ID，直接用语义预测填充未分配的位置
for cls in stuff_class_idxs:
    is_stuff = (semantic_pred == cls)
    is_unassigned = (panoptic_seg == 0)
    panoptic_seg[is_stuff & is_unassigned] = cls  # 用类别 ID 标记
```

### 最终输出

```python
# 每条线段的最终结果
panoptic_seg  # (N,)  整数，编码方式：
              #   thing 类：instance_id（唯一整数，可反查类别）
              #   stuff 类：cls_id × 1000（区分 stuff 和 thing）
```

## 9.6 第四步：Vector NMS

**代码位置**：`model/vecformer/modules/` → `vector_nms()`

在输出最终结果前，去除重复的实例预测：

```python
# 两个实例预测的掩码 IoU 超过阈值时，保留置信度更高的，去除低的
for i in range(len(predictions)):
    for j in range(i+1, len(predictions)):
        iou = mask_iou(masks[i], masks[j])
        if iou > nms_threshold:
            # 保留置信度高的
            if scores[i] > scores[j]:
                remove[j] = True
            else:
                remove[i] = True

final_predictions = [p for i, p in enumerate(predictions) if not remove[i]]
```

## 9.7 完整推理调用链

```python
# 用户调用
trainer.evaluate()
        │
        ▼ VecFormerTrainer.prediction_step()
        │   with torch.no_grad():
        │       outputs = model(**batch)
        │
        ▼ VecFormer.forward()（推理分支）
        │   backbone_feats = self.backbone(data_dict)     # (N, 64)
        │   backbone_feats = self.lfe(backbone_feats)     # (N, 64)
        │   queries = backbone_feats                      # Q=N，用全部特征
        │   cad_outputs = self.cad_decoder(backbone_feats, queries)
        │   return self.predict(cad_outputs)
        │
        ▼ VecFormer.predict()
        │   pred_sem, sem_scores = predict_semantic(pred_sem_labels)
        │   inst_masks, inst_classes, inst_scores = predict_instance(...)

        │       dict_pred_sem_segs=...,
```

## 9.8 理解输出的实际含义

对一张有 800 条线段的图纸，完整推理后：

```python
dict_pred_panop_segs = {
    "data_path": "path/to/floor_plan.json",
    "pred_panop_seg": tensor([
        #  线段索引 → (类别, 实例ID)
        # 0: 门实例#1 的线段
        3_001,    # 类别3（门），实例001
        3_001,
        3_002,    # 类别3（门），实例002（另一扇门）
        7_003,    # 类别7（窗），实例003
        12_000,   # 类别12（墙），stuff类，实例000（统一编号）
        ...
    ])
}
```

这个输出可以用来：
- 统计每种类型有多少个实例（数门、数窗）
- 高亮特定实例（找到特定编号的门）
- 计算 PQ 指标（与真实标注对比）
