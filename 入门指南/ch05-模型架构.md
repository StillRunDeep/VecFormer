## 5.1 整体架构图

```
输入（一个 batch，B 张图，总计 N 条线段）
  coords:    (N, 4)   每条线的端点坐标
  feats:     (N, 7)   每条线的几何+颜色特征
  cu_seqlens:(B+1,)   批次边界索引
  prim_ids:  (N,)     图元类型
  layer_ids: (N,)     CAD 图层
      │
      ▼
┌──────────────────────────────────────────────────────┐
│  PointTransformerV3 骨干网络                          │
│  ─────────────────────────────────────────────────   │
│  编码器（5层，逐步下采样）：                            │
│    层1: (N,  7)→(N,   32)  stride=1                  │
│    层2: (N,  32)→(N/2, 64)  stride=2（下采样）        │
│    层3: (N/4, 128)          stride=2                  │
│    层4: (N/8, 256)          stride=2 （depth=6，最深）│
│    层5: (N/16,512)          stride=2                  │
│                                                      │
│  解码器（4层，逐步上采样）：                            │
│    层1: (N/16,512)→(N/8, 256)  + skip connection      │
│    层2: (N/8, 256)→(N/4, 128)  + skip connection      │
│    层3: (N/4, 128)→(N/2,  64)  + skip connection      │
│    层4: (N/2,  64)→(N,    64)  + skip connection      │
│                                                      │
│  输出：(N, 64)  每条线段的语义特征向量                  │
└──────────────────┬───────────────────────────────────┘
                   │ (N, 64)
                   ▼
┌──────────────────────────────────────────────────────┐
│  Layer Fusion（图层融合，可选）                        │
│  同一 CAD 图层的线段特征聚合，增强上下文               │
└──────────────────┬───────────────────────────────────┘
                   │ (N, 64)
                   ▼
┌──────────────────────────────────────────────────────┐
│  CAD Decoder（解码器，6个 Block）                      │
│  ─────────────────────────────────────────────────   │
│  初始查询向量: (Q, 64)  Q≈N（推理时），训练时按阈值采样 │
│                                                      │
│  Block × 6：                                         │
│    ① 自注意力：查询向量互相交流                        │
│    ② 交叉注意力：查询向量从骨干特征提取信息             │
│    ③ FFN：前馈网络                                    │
│    ④ 预测头：输出本块的实例/语义预测                    │
└──────────────────┬───────────────────────────────────┘
                   │
          ┌────────┴──────────┐
          ▼                   ▼
   语义预测                 实例预测
  (N, 35)                 (Q, N) 掩码
  每条线的类别              + (Q, 35) 类别
                           + (Q, 1)  置信度
```

## 5.2 骨干网络：PointTransformerV3

**代码位置**：`model/vecformer/point_transformer_v3/model.py`

### 5.2.1 为什么用 PointTransformerV3？

PointTransformerV3（PTv3）是一个专为点云设计的骨干网络，有三个关键优势：

1. **稀疏卷积**：只在有线段的位置计算，空白区域跳过
2. **序列化局部注意力**：不做全局注意力（O(N²)太慢），而是局部窗口注意力（O(N×K)）
3. **多尺度特征**：编码器-解码器结构提取不同粒度的特征

### 5.2.2 序列化注意力（Serialized Attention）

```
核心问题：N 条线段做全局注意力 → N² 次计算，N=5000 时有 2500 万次！

解决方案：
  Step 1：把 N 条线段按 Hilbert 曲线排序
           空间上相邻的线段，在序列中也相邻

  Step 2：把排好序的线段切成若干"patch"，每个 patch 1024 条
           N=5000 → 5 个 patch

  Step 3：每个 patch 内部做注意力（1024² = 100 万次，小多了）

  Step 4：重复 4 种排序方式（z, z-trans, hilbert, hilbert-trans）
           弥补不同排序方式在边界处的"注意力断层"
```

```python
# 来自 configuration_vecformer.py
backbone_config = dict(
    order=("z", "z-trans", "hilbert", "hilbert-trans"),  # 4种排序
    enc_patch_size=(1024, 1024, 1024, 1024, 1024),        # 每patch 1024条线
    enc_depths=(2, 2, 2, 6, 2),   # 各层 Transformer Block 数量（第4层最深）
    enc_channels=(32, 64, 128, 256, 512),  # 各层特征维度
    stride=(2, 2, 2, 2),          # 各层下采样倍率
)
```

### 5.2.3 Skip Connection（跳跃连接）

解码器在上采样时，把对应编码器层的特征拼接进来，保留低级细节：

```
编码器层3输出: (N/4, 128) ──────────────────┐
                                             │（拼接）
解码器层2输入: (N/8, 256) → 上采样 → (N/4, 256+128=384) → Linear → (N/4, 128)
```

### 5.2.4 Flash Attention

在 `SerializedAttention` 内部，注意力计算有两条路径：

```python
if self.enable_flash:
    # 路径A：Flash Attention（快，需要 sm_75+ GPU）
    feat = flash_attn.flash_attn_varlen_qkvpacked_func(
        qkv.to(torch.float16), ...
    )
else:
    # 路径B：标准注意力（慢但兼容所有 GPU）
    attn = (q * scale) @ k.T
    attn = softmax(attn)
    feat = attn @ v
```

如果 `flash_attn` 未安装，会自动 fallback 到路径 B 并打印警告（这是我们在第 0 章修复的内容）。

## 5.3 图层融合：FusionLayerFeatsModule

**代码位置**：`model/vecformer/modules/fusion_layer_feats_module.py`

CAD 图纸有图层（Layer）概念——建筑师把门放在"门图层"，把墙放在"墙图层"。同一图层内的线段具有相似的语义，值得相互交流信息。

```
输入：骨干特征 (N, 64) + layer_ids (N,)

GroupFeatFusion 操作：
  1. 按 layer_id 分组
  2. 每组内的特征取平均（group mean pooling）
  3. 把组平均特征广播回每条线段
  4. 与原特征拼接/相加

输出：(N, 64)  形状不变，但融入了图层级上下文
```

## 5.4 CAD 解码器

**代码位置**：`model/vecformer/cad_decoder/cad_decoder.py`

### 5.4.1 查询向量机制（来自 DETR）

DETR（Detection Transformer）的核心思想：用一组**可学习的查询向量**代表"候选实例"，通过注意力机制从特征中提取各自关心的信息。

```
传统方法：滑动窗口扫描图像，找候选框
DETR 方法：N 个查询向量同时"询问"全图，各自找自己对应的目标
```

```python
# 查询向量初始化
# 训练时：从骨干特征中采样（置信度 > query_thr=0.5 的位置）
# 推理时：所有线段都作为查询向量（Q=N）
queries = backbone_feats[selected_indices]  # (Q, 64)
```

### 5.4.2 单个解码器 Block

```
输入：queries (Q, 64)  +  backbone_feats (N, 64)
  │
  ├─ [自注意力 Self-Attn]
  │   queries 内部互相关注，避免多个查询预测同一实例
  │   输出: (Q, 64)
  │
  ├─ Add & LayerNorm
  │
  ├─ [交叉注意力 Cross-Attn]
  │   queries 作为 Query，backbone_feats 作为 Key/Value
  │   每个 query 从全局特征中"提取"自己关心的信息
  │   输出: (Q, 64)
  │
  ├─ Add & LayerNorm
  │
  ├─ [FFN]
  │   Linear(64→256)→GELU→Linear(256→64)
  │   输出: (Q, 64)
  │
  └─ Add & LayerNorm → 输出 queries: (Q, 64)
```

### 5.4.3 迭代预测（iter_pred=True）

每个 block 都会输出一次预测：

```
Block 1 → 预测1（粗糙）
Block 2 → 预测2（稍好）
...
Block 6 → 预测6（最精确）

训练时：对所有6个预测都计算损失，加权求和
  loss = Σᵢ weight_i × loss_i
  好处：更强的监督信号，更快收敛

推理时：只用最后一块（Block 6）的预测
```

### 5.4.4 预测头

每个 block 末尾的预测头：

```python
# 语义预测（每条线段）
pred_sem_labels = Linear(64, 35)(backbone_feats)  # (N, 35) logits

# 实例预测（每个查询向量）
pred_inst_labels = Linear(64, 35)(queries)         # (Q, 35) 类别 logits
pred_inst_scores = Linear(64, 1)(queries)          # (Q, 1)  置信度
pred_inst_masks  = queries @ backbone_feats.T      # (Q, N)  掩码 logits
                   # 点积：query 和每条线段的特征相似度
```

## 5.5 损失函数

**代码位置**：`model/vecformer/criterion/`

### 5.5.1 匈牙利匹配

训练时，模型输出 Q 个预测实例，真实标注有 M 个实例，需要找最优配对：

```python
# 代码位置：instance_criterion.py

# 计算代价矩阵（Q × M）
cost_class = -pred_inst_labels[:, target_labels]   # 类别代价
cost_mask  = compute_bce_dice_cost(pred_masks, target_masks)  # 掩码代价
cost = cost_class + cost_mask  # (Q, M)

# 用匈牙利算法找最优匹配
row_idx, col_idx = linear_sum_assignment(cost.cpu().numpy())
# row_idx[i] 号预测 ↔ col_idx[i] 号真实标注
```

### 5.5.2 实例损失

对匹配后的配对计算损失：

```
类别损失（Class Loss）：
  对 Q 个查询向量预测类别
  CE loss，权重：
    - 匹配到真实实例的查询：正常权重 2.5
    - 未匹配的查询：权重 0.01（预期预测"无对象"类别）
  Label smoothing=0.1（防止过拟合）

掩码损失（Mask Loss）：
  BCE Loss  = 逐元素二元交叉熵
  Dice Loss = 1 - 2×|预测∩真实| / (|预测|+|真实|)
  两者都乘以权重 5.0

置信度损失（Score Loss）：
  预测的置信度分数应与预测掩码的 IoU 对齐
  Loss = MSE(pred_score, mask_iou)
  权重 0.5
```

### 5.5.3 语义损失

对每条线段独立预测类别：

```
对 N 条线段，每条预测 35 类概率
Cross Entropy Loss，权重：
  - 前景类（0–29）：1.0
  - 背景类（30–34）：0.1（背景线段多，降低其权重防止压制前景）
Label smoothing=0.1
```

### 5.5.4 总损失

```
total_loss = instance_loss + semantic_loss

实例损失包含：
  = class_loss × 2.5
  + bce_loss × 5.0
  + dice_loss × 5.0
  + score_loss × 0.5
```

## 5.6 `VecFormerConfig` 中各参数的含义

**代码位置**：`model/vecformer/configuration_vecformer.py`

| 参数 | 默认值 | 含义 |
|------|-------|------|
| `num_instance_classes` | 35 | 实例类别数（包含 thing+stuff）|
| `thing_class_idxs` | [0–29] | 需要区分实例的类别 |
| `stuff_class_idxs` | [30–34] | 背景类（不区分实例）|
| `use_layer_fusion` | True | 是否启用图层融合 |
| `sample_mode` | "line" | 线段模式（7维特征）或点模式（4维）|
| `query_thr` | 0.5 | 训练时采样查询向量的阈值 |
| `max_num_queries` | -1 | 最大查询向量数（-1表示无限制）|
| `enable_flash` | True | 是否启用 Flash Attention |
| `enc_depths` | (2,2,2,6,2) | 各编码器层的 Block 数量 |
| `enc_channels` | (32,64,128,256,512) | 各编码器层的通道数 |
| `n_blocks` | 6 | CAD 解码器的 Block 数量 |
| `iter_pred` | True | 每个解码器 Block 都输出预测 |
| `num_topk_preds` | 600 | 推理时保留的最大实例数 |
| `pred_score_thr` | 0.5 | 置信度过滤阈值 |
| `mask_logit_thr` | 0.3 | 掩码二值化阈值 |
