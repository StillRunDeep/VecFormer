## 4.1 数据从哪里来

### 原始数据集：FloorPlanCAD

- **格式**：SVG 矢量图形，每张图是一个 `.svg` 文件
- **内容**：建筑平面图，包含墙、门、窗、家具等符号
- **标注**：每条线段有语义类别 ID 和实例 ID（由人工标注）
- **下载**：通过 Google Drive（见 README.md）

### 预处理：SVG → JSON

原始 SVG 文件不能直接训练，需要先运行预处理脚本提取线段信息：

```bash
python data/floorplancad/preprocess.py \
    --input_dir=datasets/FloorPlanCAD \
    --output_dir=datasets/FloorPlanCAD-sampled-as-line-jsons \
    --dynamic_sampling \
    --connect_lines
```

每张图生成一个 `.json` 文件，存储所有线段的坐标、颜色、标签等。

## 4.2 原始数据结构：`SVGData`

**代码位置**：`data/floorplancad/dataclass_define.py`

```python
@dataclass
class SVGData:
    viewBox: list[float]           # 图纸边界 [minx, miny, width, height]
    coords: list[list[float]]      # 每条线段的坐标 [[x1,y1,x2,y2], ...]
    colors: list[list[int]]        # 每条线段的颜色 [[R,G,B], ...]
    widths: list[float]            # 每条线段的线宽 [w1, w2, ...]
    primitive_ids: list[int]       # 图元类型 ID（同一个符号的线段共享 ID）
    layer_ids: list[int]           # CAD 图层 ID（建筑师用图层组织线段）
    semantic_ids: list[int]        # 语义类别标签（门=0, 窗=1, 墙=12...）
    instance_ids: list[int]        # 实例 ID（第1扇门=0, 第2扇门=1...）
    primitive_lengths: list[float] # 每条线段的欧氏距离长度
```

**一个小例子**（3 条线段，构成一扇门）：

```python
SVGData(
    viewBox=[0, 0, 100, 80],
    coords=[[10,20,10,50], [10,50,30,50], [30,50,30,20]],  # 门框的三条边
    colors=[[0,0,0], [0,0,0], [0,0,0]],                    # 都是黑色
    widths=[1.0, 1.0, 1.0],
    primitive_ids=[5, 5, 5],          # 同属于"图元5"（这扇门）
    layer_ids=[2, 2, 2],              # 都在图层2
    semantic_ids=[3, 3, 3],           # 语义类别3 = 门
    instance_ids=[0, 0, 0],           # 第0个门实例
    primitive_lengths=[30.0, 20.0, 30.0],
)
```

## 4.3 模型输入结构：`VecData`

`SVGData` 是纯 Python 列表，`VecData` 是转换后的 PyTorch 张量，可以直接送入模型。

**代码位置**：`data/floorplancad/dataclass_define.py`

```python
@dataclass
class VecData:
    data_path: str                 # 源文件路径（用于调试/可视化）

    # ─── 输入特征 ───────────────────────────────────────────────
    coords: torch.Tensor           # 坐标，Line模式: (N, 4)  [x1,y1,x2,y2]
                                   #         Point模式: (N, 2) [cx, cy]

    feats: torch.Tensor            # 特征，Line模式:  (N, 7)
                                   #   [length, |dx|, |dy|, cx, cy, r/255, g/255]
                                   #         Point模式: (N, 4)
                                   #   [cx, cy, r/255, g/255]

    prim_ids: torch.Tensor         # (N,) 图元类型 ID（整数）
    layer_ids: torch.Tensor        # (N,) 图层 ID（整数）

    # ─── 标签 ────────────────────────────────────────────────────
    sem_ids: torch.Tensor          # (N,) 语义类别 ID
    inst_ids: torch.Tensor         # (N,) 实例 ID
    prim_lengths: torch.Tensor     # (N,) 线段长度（用于加权评估）
```

## 4.4 Line 模式下特征的含义

**为什么是 7 维？**（Line 模式，`sample_mode="line"`）

```
每条线段的完整特征向量（7维）：

feats[i] = [
    length,      # 0: 线段长度 = sqrt((x2-x1)² + (y2-y1)²)
    abs(dx),     # 1: 水平跨度（绝对值）= |x2 - x1|
    abs(dy),     # 2: 垂直跨度（绝对值）= |y2 - y1|
    cx,          # 3: 中心点 x = (x1+x2)/2
    cy,          # 4: 中心点 y = (y1+y2)/2
    r / 255,     # 5: 颜色 R 通道（归一化到 [0,1]）
    g / 255,     # 6: 颜色 G 通道（归一化到 [0,1]）
]
```

注意：颜色只用了 R 和 G（没有 B）。这是因为在 CAD 绘图中，蓝色 (B) 通常可以由 R 和 G 推导，减少一维特征不影响表达能力。

**坐标（4维）**：

```
coords[i] = [x1, y1, x2, y2]   # 线段两端点的绝对坐标

坐标会被归一化到 [-0.5, 0.5]：
  norm_x = (x - viewBox[0]) / viewBox[2] - 0.5
  norm_y = (y - viewBox[1]) / viewBox[3] - 0.5
```

## 4.5 数据加载流程

**代码位置**：`data/floorplancad/floorplancad.py`

```python
class FloorPlanCAD(Dataset):
    def __getitem__(self, idx) -> VecData:
        # 步骤1：读 JSON 文件
        with open(self.data_paths[idx]) as f:
            raw = json.load(f)
        svg_data = SVGData(**raw)

        # 步骤2：转成张量
        svg_tensor = to_tensor(svg_data)      # list → torch.Tensor

        # 步骤3：归一化坐标到 [-0.5, 0.5]
        svg_tensor.coords = normalize_coords(
            svg_tensor.coords,
            svg_tensor.viewBox
        )

        # 步骤4：数据增强（只在训练时）
        if self.split == "train":
            svg_tensor.coords = augment(
                svg_tensor.coords,
                flip_prob=0.5,       # 50% 概率水平翻转
                scale_range=[0.5, 1.5],  # 随机缩放
                translate=[0.5, 0.5],    # 随机平移
            )

        # 步骤5：提取线段特征（coords → feats）
        vec_data = to_vec_data(svg_tensor)

        return vec_data
```

## 4.6 数据增强

**代码位置**：`data/floorplancad/augment_utils.py`

数据增强在训练时随机变换图纸，让模型对各种情况更鲁棒。

| 增强方式 | 参数 | 实现方式 |
|---------|------|---------|
| 水平翻转 | `random_horizontal_flip=0.5` | x → -x，然后重新归一化 |
| 垂直翻转 | `random_vertical_flip=0.5` | y → -y，然后重新归一化 |
| 随机缩放 | `random_scale=[0.5, 1.5]` | 坐标乘以随机缩放因子 |
| 随机平移 | `random_translation=[0.5, 0.5]` | 坐标加上随机偏移量 |
| 随机旋转 | `random_rotate=False`（默认关闭）| 坐标绕中心旋转任意角度 |

```python
# 翻转的效果（示意）
# 原始：门在左下角
coords_original = [[10, 50, 30, 50]]   # 一条横线

# 水平翻转后：门在右下角（x 坐标取反）
coords_flipped = [[-10, 50, -30, 50]]  # → 归一化后重新映射
```

**评估时不做增强**（`eval_transform_args` 所有概率设为 0），确保结果可复现。

## 4.7 批次拼接：`collate_fn`

这是理解本项目数据处理的**最关键函数**。

**问题**：不同图纸的线段数量不同（有的 200 条，有的 5000 条），PyTorch 默认的 `DataLoader` 无法把它们拼成一个 batch（因为形状不同）。

**解决方案**：把所有图纸的线段**直接拼接**成一个长序列，用 `cu_seqlens` 记录每张图的边界。

```python
@staticmethod
def collate_fn(batch: list[VecData]) -> dict:
    # batch = [VecData(图1), VecData(图2), VecData(图3)]
    # 假设三张图分别有 300, 500, 200 条线段

    # 1. 拼接坐标
    coords = torch.cat([item.coords for item in batch], dim=0)
    # 形状: (300+500+200, 4) = (1000, 4)

    # 2. 拼接特征
    feats = torch.cat([item.feats for item in batch], dim=0)
    # 形状: (1000, 7)

    # 3. 计算边界索引（累计序列长度）
    lengths = [len(item.coords) for item in batch]  # [300, 500, 200]
    cu_seqlens = torch.tensor([0] + list(accumulate(lengths)))
    # cu_seqlens = [0, 300, 800, 1000]

    # 4. 拼接标签
    sem_ids    = torch.cat([item.sem_ids    for item in batch])  # (1000,)
    inst_ids   = torch.cat([item.inst_ids   for item in batch])  # (1000,)
    prim_ids   = torch.cat([item.prim_ids   for item in batch])  # (1000,)
    layer_ids  = torch.cat([item.layer_ids  for item in batch])  # (1000,)

    return {
        "coords": coords,          # (1000, 4)
        "feats": feats,            # (1000, 7)
        "cu_seqlens": cu_seqlens,  # [0, 300, 800, 1000]
        "prim_ids": prim_ids,      # (1000,)
        "layer_ids": layer_ids,    # (1000,)
        "sem_ids": sem_ids,        # (1000,)
        "inst_ids": inst_ids,      # (1000,)
        ...
    }
```

### `cu_seqlens` 的用法

```python
# 取第 i 张图的数据（i 从 0 开始）
def get_item_i(packed_tensor, cu_seqlens, i):
    start = cu_seqlens[i]
    end   = cu_seqlens[i + 1]
    return packed_tensor[start:end]

# 例：取第1张图（索引i=1）的特征
feats_img1 = feats[cu_seqlens[1]:cu_seqlens[2]]
# feats[300:800]，形状 (500, 7)
```

## 4.8 标签的组织方式

`inst_ids` 里的值是原始实例 ID（在每张图内部唯一），但不同图的实例 ID 可能重复（图1和图2都有"实例0"）。

模型内部用 `cu_numprims`（每张图有多少条线段）来解决这个问题，在 `prepare_targets()` 里会把实例 ID 转成**每张图独立的实例掩码矩阵**。

## 4.9 数据的完整生命周期

```
① 磁盘上的 JSON 文件
       ↓ FloorPlanCAD.__getitem__()
② 内存中的 VecData（单张图）
  - coords: (N, 4)
  - feats:  (N, 7)
  - sem_ids, inst_ids, ...
       ↓ collate_fn()
③ 一个 batch 的字典（多张图拼接）
  - coords:     (N_total, 4)
  - feats:      (N_total, 7)
  - cu_seqlens: (B+1,)
       ↓ 送入 VecFormer.forward()
④ 骨干网络特征
  - backbone_feats: (N_total, 64)
       ↓ CAD Decoder
⑤ 预测结果
  - pred_sem_labels:  (N_total, 35)
  - pred_inst_masks:  (Q, N_total)
       ↓ Criterion / Evaluator
⑥ 损失值 / 评估指标
```
