## 3.2 程序入口：`launch.py`

**它做什么**：把配置、模型、数据集、训练器组装在一起，启动整个流程。

```python
# launch.py 的核心逻辑（简化版）

def main():
    # 1. 解析参数（从 yaml 文件 + 命令行）
    training_args = TrainingArguments(...)

    # 2. 构建模型（通过注册表）
    model, TrainerClass = build_model(training_args.model_args_path)

    # 3. 构建数据集（通过注册表）
    train_dataset, eval_dataset, collate_fn = build_dataset(...)

    # 4. 创建训练器
    trainer = TrainerClass(
        model=model,
        args=training_args,
        train_dataset=train_dataset,
        eval_dataset=eval_dataset,
        data_collator=collate_fn,
    )

    # 5. 执行（根据 launch_mode 决定做什么）
    if launch_mode == "train":
        trainer.train()
    elif launch_mode == "test":
        trainer.evaluate()
    elif launch_mode == "continue":
        trainer.train(resume_from_checkpoint=checkpoint_path)
```

**关键点**：`launch.py` 本身很薄，它只是把各模块拼接起来。真正的逻辑在 `model/` 和 `data/` 里。

## 3.3 配置目录：`configs/`

```
configs/
├── vecformer.yaml           ← 训练超参数（学习率、batch大小、训练轮数…）
├── model/
│   └── vecformer.yaml       ← 模型配置（指定用哪个模型）
└── data/
    └── floorplancad.yaml    ← 数据集配置（数据路径、增强参数…）
```

**三个配置文件各司其职**：

```yaml
# configs/vecformer.yaml —— 训练过程
num_train_epochs: 500
per_device_train_batch_size: 2
learning_rate: 0.0001
metric_for_best_model: PQ      # 用全景质量选最优模型

# configs/model/vecformer.yaml —— 模型选择
model_name: vecformer          # 告诉 build_model() 用哪个模型

# configs/data/floorplancad.yaml —— 数据路径和增强
dataset_name: floorplancad
dataset_args:
    root_dir: datasets/FloorPlanCAD-sampled-as-line-jsons
```

**为什么分三个文件**：方便复用——同一个数据集可以搭配不同模型，同一个模型可以用不同训练配置。

## 3.4 运行脚本：`scripts/`

```
scripts/
├── train.sh      ← 从零开始训练
├── test.sh       ← 只做评估（加载已有模型权重）
├── resume.sh     ← 断点续训（保留优化器状态）
└── continue.sh   ← 继续训练（加载权重但重置优化器）
```

**train.sh 核心内容**：

```bash
# 用 torchrun 启动多 GPU 分布式训练
torchrun \
    --nproc_per_node=8 \        # 使用 8 块 GPU
    launch.py \
    --launch_mode train \
    --config_path configs/vecformer.yaml \
    --model_args_path configs/model/vecformer.yaml \
    --data_args_path configs/data/floorplancad.yaml
```

**resume.sh vs continue.sh 的区别**：

| | resume.sh | continue.sh |
|---|---|---|
| 加载模型权重 | ✅ | ✅ |
| 加载优化器状态 | ✅（完全恢复）| ❌（重置）|
| 学习率调度器 | 从中断处继续 | 从头开始 |
| 适用场景 | 意外中断后恢复 | 在已训练模型上进行新一轮调优 |

**数据文件的生命周期**：

```
原始数据（SVG 文件）
      ↓ 运行 preprocess.py（一次性）
JSON 文件（每张图一个 .json）
      ↓ 运行时 floorplancad.py 读取
内存中的 VecData 对象（Tensor）
      ↓ collate_fn 拼成 batch
模型的输入字典
```

```
utils/

    ├── printer_callback_patch.py  ← 修改进度条显示

```

**猴子补丁（Monkey Patch）**：在不修改第三方库源码的前提下，运行时替换其部分函数行为。本项目用它来定制 HuggingFace Trainer 的日志和进度显示。

## 3.8 注册表机制（理解代码组织的关键）

项目通过"注册表"实现配置与代码解耦。

```python
# model/__init__.py
_model_registry = {}

def register_model(name):
    def decorator(build_fn):
        _model_registry[name] = build_fn
        return build_fn
    return decorator

def build_model(model_args_path):
    config = load_yaml(model_args_path)   # {"model_name": "vecformer", ...}
    name = config["model_name"]
    return _model_registry[name](config["model_args"])
```

```python
# model/vecformer/__init__.py
@register_model("vecformer")            # 把字符串 "vecformer" 注册进去
def build(model_args):
    return VecFormer(VecFormerConfig(**model_args)), VecFormerTrainer
```

**效果**：`launch.py` 只需读配置文件里的字符串 `"vecformer"`，就能找到对应的模型类，无需 import 具体类。添加新模型时，只需新建一个文件并加 `@register_model("my_model")`，无需修改 `launch.py`。
