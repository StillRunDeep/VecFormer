## 6.1 从命令到训练的完整路径

```
bash scripts/train.sh
        │
        ▼
torchrun --nproc_per_node=8 launch.py --launch_mode train ...
        │
        ▼ dist.init_process_group("nccl")  启动 8 个进程，各占一块 GPU
        │
        ▼ apply_patches()  打上 HuggingFace 补丁
        │
        ▼ 解析配置
        │   training_args = TrainingArguments(configs/vecformer.yaml)
        │   model_args    = load_yaml(configs/model/vecformer.yaml)
        │   data_args     = load_yaml(configs/data/floorplancad.yaml)
        │
        ▼ 构建模型
        │   config = VecFormerConfig(**model_args)
        │   model  = VecFormer(config)
        │   （约 100M 参数，自动分配到各 GPU）
        │
        ▼ 构建数据集
        │   train_dataset = FloorPlanCAD(split="train", ...)
        │   eval_dataset  = FloorPlanCAD(split="val",   ...)
        │
        ▼ 创建训练器
        │   trainer = VecFormerTrainer(
        │       model=model,
        │       args=training_args,
        │       train_dataset=train_dataset,
        │       eval_dataset=eval_dataset,
        │       data_collator=FloorPlanCAD.collate_fn,
        │   )
        │
        ▼ trainer.train()
           （进入 HuggingFace Trainer 的训练循环）
```

## 6.2 训练循环详解

HuggingFace Trainer 的训练循环（简化版）：

```python
for epoch in range(num_train_epochs):          # 500 轮
    for batch in train_dataloader:             # 每轮遍历所有训练数据

        # ① 前向传播（VecFormerTrainer.compute_loss 里）
        outputs = model(**batch)
        loss = outputs.loss                    # 标量，包含实例+语义损失

        # ② 反向传播
        loss.backward()                        # 计算所有参数的梯度

        # ③ 梯度裁剪（防止梯度爆炸）
        torch.nn.utils.clip_grad_norm_(model.parameters(), max_grad_norm=1.0)

        # ④ 参数更新
        optimizer.step()
        lr_scheduler.step()
        optimizer.zero_grad()

        # ⑤ 定期评估（每 eval_steps=2175 步）
        if global_step % eval_steps == 0:
            metrics = evaluate()               # 计算 PQ 等指标
            if metrics["PQ"] > best_pq:
                save_checkpoint()              # 保存最优模型

        # ⑥ 定期日志（每 logging_steps=200 步）
        if global_step % logging_steps == 0:
            log(loss, lr, ...)
```

## 6.3 优化器：AdamW

```yaml
# configs/vecformer.yaml
optim: adamw_torch_fused      # 使用融合版 AdamW（更快）
learning_rate: 0.0001         # 初始学习率 1e-4
weight_decay: 0.05            # L2 正则化系数
```

**AdamW 是什么**：Adam 优化器 + 权重衰减（Weight Decay）。

- **Adam**：自适应学习率，对不同参数用不同步长（比 SGD 收敛快）
- **Weight Decay**：每步更新时给参数乘以一个小于 1 的系数，防止参数过大（过拟合）

---

## 6.4 学习率调度：warmup_stable_decay

```yaml
lr_scheduler_type: warmup_stable_decay
warmup_ratio: 0.05            # 前 5% 的步骤做 warmup
lr_scheduler_kwargs:
    num_decay_steps: 43500    # 最后 43500 步做衰减
    min_lr_ratio: 0.1         # 最终学习率 = 初始 × 0.1
```

**学习率变化曲线**：

```
学习率
  ^
  |      _______________
  |    /                \____
  |  /                       \___
  |/                              \___ → 0.0001 × 0.1 = 0.00001
  +------------------------------------------→ 训练步骤
  |  ^warmup  ^稳定阶段          ^衰减阶段^

  warmup 阶段：从 0 线性增加到 0.0001（防止初始参数大更新导致不稳定）
  稳定阶段：保持 0.0001
  衰减阶段：最后 43500 步从 0.0001 衰减到 0.00001
```

## 6.5 多 GPU 分布式训练

**代码位置**：`launch.py`，启动方式：`torchrun`

### DDP（DistributedDataParallel）原理

```
8块 GPU，每块独立运行一个进程：

  GPU 0: batch[0:2]  → forward → loss0 → grad0
  GPU 1: batch[2:4]  → forward → loss1 → grad1
  ...
  GPU 7: batch[14:16]→ forward → loss7 → grad7

  All-Reduce（NCCL）：
    梯度求平均 → grad_avg = (grad0 + ... + grad7) / 8

  每块 GPU 用 grad_avg 更新参数（保持同步）
```

**关键参数**：

```bash
# scripts/train.sh
NPROC_PER_NODE=8     # 每台机器 8 块 GPU
NNODES=1             # 1 台机器
# 总 batch size = 8 × per_device_batch_size(2) = 16
```

**单 GPU 运行（用于调试）**：

```bash
NPROC_PER_NODE=1 bash scripts/train.sh
# 注意：单 GPU 时 batch size 只有 2，需要更多轮数才收敛
```

## 6.6 评估流程

每隔 `eval_steps=2175` 步，暂停训练做一次验证：

```python
# VecFormerTrainer.prediction_step（简化版）

with torch.no_grad():          # 关闭梯度（节省内存）
    outputs = model(**batch)   # 推理模式（model.eval()）

# outputs.metric_states 包含：
#   tp_per_class:       每类 TP 数量
#   fp_per_class:       每类 FP 数量
#   fn_per_class:       每类 FN 数量
#   tp_iou_per_class:   每类 TP 的 IoU 之和
```

所有 batch 的 metric_states 汇总后，计算最终 PQ：

```python
# evaluator.py
PQ = {}
for cls in range(num_classes):
    tp = tp_per_class[cls]
    fp = fp_per_class[cls]
    fn = fn_per_class[cls]
    sq = tp_iou_per_class[cls] / tp   # 平均 IoU
    rq = tp / (tp + 0.5*fp + 0.5*fn) # F1-score
    PQ[cls] = sq * rq

mean_PQ = mean(PQ.values())           # 所有类别的平均 PQ
```

## 6.7 检查点保存策略

```yaml
metric_for_best_model: PQ   # 用验证集 PQ 决定是否保存
save_strategy: best          # 只保存最优的模型（不是每 N 步都保存）
save_total_limit: 5          # 最多保留 5 个检查点
```

**存档目录结构**：

```
outputs/20240101_120000/
├── checkpoint-best/          # 当前最优模型
│   ├── config.json           # VecFormerConfig（超参数）
│   ├── model.safetensors     # 模型权重
│   ├── optimizer.pt          # 优化器状态（resume 时用）
│   ├── scheduler.pt          # 学习率调度器状态
│   └── trainer_state.json    # 训练进度信息
├── runs/                     # TensorBoard 日志
└── train_results.json        # 最终训练结果
```

## 6.8 不同启动模式的区别

| 启动脚本 | `launch_mode` | 加载权重 | 加载优化器 | 说明 |
|---------|---------------|---------|-----------|------|
| `train.sh` | `train` | ❌ | ❌ | 从零开始 |
| `resume.sh` | `train` + checkpoint | ✅ | ✅ | 完全断点续训 |
| `continue.sh` | `continue` | ✅ | ❌ | 新一轮训练，重置优化器 |
| `test.sh` | `test` | ✅ | ❌ | 只做评估，不更新参数 |

```python
# launch.py 中的模式切换逻辑
if training_args.launch_mode == "train":
    trainer.train()
elif training_args.launch_mode == "continue":
    trainer.train(resume_from_checkpoint=ckpt_path)
    # HuggingFace Trainer 会加载模型权重，但不加载优化器
elif training_args.launch_mode == "test":
    trainer.evaluate()
```

## 6.9 监控训练过程

### TensorBoard

```bash
# 启动 TensorBoard
tensorboard --logdir outputs/

# 浏览器打开 http://localhost:6006
```

关键曲线：
- `train/loss`：总损失（应持续下降）
- `eval/PQ`：验证集全景质量（越高越好，用于选最优模型）
- `eval/SQ`：分割质量
- `eval/RQ`：识别质量
- `train/learning_rate`：学习率变化（验证 warmup/decay 是否正确）
