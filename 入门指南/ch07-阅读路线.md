## 整体路线概览

```
阶段一：数据（2–3 天）
  → 搞清楚数据长什么样、怎么加载

阶段二：配置（1 天）

阶段三：基础组件（3–5 天）
  → 搞清楚注意力、FFN 等积木块
      ↓
阶段四：骨干网络（5–7 天）
  → 搞清楚特征提取过程
      ↓
阶段五：解码与损失（5–7 天）
  → 搞清楚如何从特征变成预测、如何计算损失
      ↓
阶段六：主模型与训练（3 天）
  → 搞清楚所有组件如何串联、训练怎么跑起来
```

## 阶段三：理解基础组件（建议 3–5 天）

### 目标

读完后能回答：
- 变长注意力（Varlen Attention）和普通注意力有什么区别？
- `cu_seqlens` 如何在注意力计算中使用？
- TransformerBlock 的完整计算流程？

### 文件清单

**文件 6**：`model/vecformer/modules/attention.py`

```
阅读重点：

VarlenSelfAttention（自注意力）：
  - qkv_proj 将输入投影为 Q、K、V
  - 调用 flash_attn 或 _varlen_sdpa_qkvpacked 做注意力计算
  - 输出形状与输入相同

VarlenSelfAttentionWithRoPE（带旋转位置编码的自注意力）：
  - 在 Q、K 上施加 RoPE（Rotary Position Embedding）
  - RoPE 把位置信息编码到旋转变换中，比绝对位置编码更好

VarlenCrossAttention（交叉注意力）：
  - Q 来自 queries，K/V 来自 backbone feats
  - 用于 CAD 解码器中查询向量"问"特征图

理解要点：
  - 为什么要用 cu_seqlens 而不是普通 padding？
    → padding 到最大长度浪费内存且引入噪声
  - .to(torch.float16) 的原因？
    → flash_attn 要求 fp16/bf16 输入
  - fallback 路径是什么？
    → flash_attn 不可用时用 F.scaled_dot_product_attention
```

**文件 7**：`model/vecformer/modules/transformer_block.py`

```
阅读重点：
  - TransformerBlock 包含哪些子模块？
  - 残差连接在哪里？
  - AddNorm 的作用是什么？
```

**文件 8**：`model/vecformer/modules/group_feat_fusion.py`

```
阅读重点：
  - GroupFeatFusion 的输入输出
  - "按 group_id 聚合"的具体实现（scatter_mean）
  - 这个操作在哪里被调用？（图层融合模块）
```

```
第五段（700–900行）：
```

## 阶段五：理解解码与损失（建议 5–7 天）

### 目标

读完后能回答：
- CAD 解码器的 Block 结构？
- 匈牙利匹配是如何实现的？
- 三种损失（class / mask / score）各是什么？

### 文件清单

**文件 11**：`model/vecformer/cad_decoder/cad_decoder.py`

```
阅读重点：
  - CADDecoderBlock 的 forward：自注意力 → 交叉注意力 → FFN
  - CADDecoder 的 forward：6 个 Block 串联，iter_pred 模式
  - 预测头：sem_label_head / inst_label_head / inst_mask_head / score_head
```

**文件 12**：`model/vecformer/criterion/instance_criterion.py`

```
阅读重点：
  - matcher()：计算代价矩阵，调用 linear_sum_assignment（匈牙利算法）
  - forward()：
    a. 对每张图调用 matcher 得到匹配结果
    b. 计算 class_loss（CE）
    c. 计算 mask_loss（BCE + Dice）
    d. 计算 score_loss（MSE）
    e. 加权求和
```

**文件 13**：`model/vecformer/criterion/semantic_criterion.py`

```
阅读重点：
  - 逐线段的 CE 损失

```

**文件 14**：`model/vecformer/evaluator/evaluator.py`

```
阅读重点：
```

## 阶段六：理解主模型与训练（建议 3 天）

### 目标

读完后能回答：
- `VecFormer.forward()` 的完整计算图？
- 训练模式和推理模式有什么不同？
- HuggingFace Trainer 如何驱动整个训练？

### 文件清单

**文件 15**：`model/vecformer/modeling_vecformer.py`（最重要的文件）

```
建议分函数阅读：

prepare_targets()：
  - 将 sem_ids / inst_ids 转换为实例掩码矩阵
  - 理解从"每条线的标签"到"每个实例的掩码"的转换

_get_data_dict()：
  - 将输入张量打包成 PTv3 需要的字典格式
  - 处理 batch_idx、序列化顺序等

forward()（训练时）：
  - backbone → lfe → 查询初始化 → cad_decoder → criterion
  - 返回 VecFormerOutput(loss=..., metric_states=None)

forward()（推理时）：
  - backbone → lfe → 查询初始化（Q=N）→ cad_decoder → predict()
  - 返回 VecFormerOutput(loss=None, metric_states=..., dict_pred_*)

predict()：
  - predict_semantic() → 每条线的语义类别
  - predict_instance() → Top-K 实例预测
  - predict_panoptic() → 融合语义和实例
```

**文件 16**：`launch.py`

```
阅读重点：
  - main() 函数的完整流程
  - build_model() 和 build_dataset() 的注册表调用
  - 三种 launch_mode 的处理
```

**文件 17**：`model/vecformer/vecformer_trainer.py`

```
阅读重点：
  - compute_loss()：调用 model(**inputs) 并返回 loss
  - prediction_step()：推理并返回 metric_states
  - compute_metrics()：汇总多个 batch 的 metric_states 计算最终 PQ
```

## 推荐的并行阅读策略

每读一个新文件时，同时打开：
1. **配置文件**（`configuration_vecformer.py`）— 确认当前参数的默认值
2. **调用方**（看谁调用了这个文件的类/函数）— 理解上下文
3. **本指南对应章节**— 对照理解

```
例：阅读 SerializedAttention 时，同时对照：
  - configuration_vecformer.py 中的 enc_patch_size / enable_flash
  - Block 类（它的调用方）
  - 第 5 章 5.2 节（本指南的解释）
```
