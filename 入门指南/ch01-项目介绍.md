## 1.1 从一张建筑平面图说起

想象一下你打开 AutoCAD，看到一张建筑平面图。图里有：

- 代表**墙体**的粗线条
- 代表**门**的弧线加直线组合
- 代表**窗**的几段平行短线
- 代表**楼梯**的等间距横线
- 代表**桌椅**的矩形、曲线……

一张平面图里可能有 **数千到数万条线段**。这些线段本身只是几何图形，背后的语义（"这是一扇门"）需要人来解读。

**VecFormer 要做的事**：让计算机自动读懂这些线段，识别出每条线段的类别，以及哪些线段组成了同一个建筑符号。

---

## 1.2 任务定义：全景符号识别

这个任务的学术名称是 **Panoptic Symbol Spotting（全景符号识别）**，它包含两个子任务：

### 子任务一：语义分割（Semantic Segmentation）

给每条线段打一个类别标签。

```
输入：一张平面图的所有线段
输出：每条线段 → 类别 ID

例如：
  线段 #001 → 类别 3（门）
  线段 #002 → 类别 3（门）
  线段 #003 → 类别 7（窗）
  线段 #004 → 类别 12（墙）
```

本项目共有 **35 个类别**，包括：
- `thing` 类（0–29）：有具体实例的符号，如门、窗、桌子、马桶……
- `stuff` 类（30–34）：背景类，如墙、地板，这类不区分实例

### 子任务二：实例分割（Instance Segmentation）

在"同一类别"内，还要区分不同的个体。

```
语义分割的结果：[门, 门, 门, 门]
实例分割的结果：[门#1, 门#1, 门#2, 门#2]
                ^^^^^^^^^^^^  ^^^^^^^^^^^^
                第一扇门的线  第二扇门的线
```

### 合并：全景分割（Panoptic Segmentation）

全景分割 = 语义分割 + 实例分割，最终每条线段得到一个二元组：

```
(类别, 实例ID)

例：
  线段 #001 → (门, 实例#1)
  线段 #002 → (门, 实例#1)
  线段 #003 → (门, 实例#2)
  线段 #004 → (墙, 无实例)   ← stuff 类不区分实例
```

## 1.3 为什么这个任务难？

### 难点一：线段数量多、无固定排列

一张图可能有 5000 条线段，且没有像图像像素那样的固定网格结构。传统 CNN 无法直接处理。

### 难点二：同类符号外观相似

两扇门的线段几何形状几乎一样，只靠局部特征无法区分，需要理解**全局上下文**（这扇门旁边是走廊还是厨房？）。

### 难点三：同一符号的线段数量不固定

一扇简单的门可能只有 3 条线段，一个复杂的楼梯可能有 40 条线段。模型需要处理**可变数量的图元**。

### 难点四：stuff 类与 thing 类的处理方式不同

墙（stuff）不需要分实例，门（thing）需要分。这两类要用不同的预测策略。

## 1.4 传统方法 vs VecFormer

### 方法一：转成图片用 CNN 处理

```
CAD 线段 → 栅格化成图像（如 512×512 像素）→ CNN → 像素级预测
```

**问题**：
- 线段的精确坐标被丢失，变成模糊的像素
- 分辨率有限，细小符号难以识别
- 图像尺寸固定，但图纸大小不固定

### 方法二：把线段当点云处理

```
每条线段的中心点 → 点云 → PointNet/PointTransformer → 预测
```

**问题**：
- 只用中心点，丢失了线段的方向、长度等几何信息
- 短线段和长线段的中心点可能相同，无法区分

### VecFormer 的方法：直接处理线段序列

```
线段序列（保留端点坐标、方向、颜色等完整信息）
    → Transformer（理解全局上下文）
    → 全景预测
```

**关键差异**：
| 维度 | 图像 CNN | 点云方法 | VecFormer |
|------|---------|---------|-----------|
| 输入表示 | 像素 | 中心点 | **完整线段**（端点+方向+颜色）|
| 几何精度 | 低（栅格化损失）| 中（丢失方向）| **高**（保留所有几何信息）|
| 上下文建模 | 局部感受野 | 局部邻域 | **全局 Attention** |
| 可变长度 | 固定图像尺寸 | 较难 | **原生支持** |

## 1.5 数据集：FloorPlanCAD

本项目使用 **FloorPlanCAD** 数据集：

- **来源**：真实建筑平面图，从 SVG 格式提取
- **规模**：数千张平面图（训练/验证/测试集划分）
- **标注**：每条线段有语义类别标签和实例 ID
- **类别数**：35 类（30 thing + 5 stuff）

典型数据量：一张平面图约 200–5000 条线段。

---

## 1.6 模型名称的含义

**VecFormer** = **Vec**tor + Trans**former**

- **Vector**：输入是向量图形（线段序列），而非栅格图像
- **Transformer**：使用 Transformer 架构处理序列

---

## 1.7 核心架构鸟瞰

```
                        一张 CAD 图
                            │
              ┌─────────────▼─────────────┐
              │   数千条线段（向量图形）      │
              │   每条线：坐标 + 颜色 + 宽度  │
              └─────────────┬─────────────┘
                            │
              ┌─────────────▼─────────────┐
              │    PointTransformerV3      │
              │    （骨干特征提取网络）       │
              │                           │
              │  输入：每条线 7 维特征       │
              │  输出：每条线 64 维语义特征   │
              │                           │
              │  核心技术：                 │
              │  · 稀疏卷积（高效处理稀疏数据）│
              │  · Hilbert 曲线序列化       │
              │  · 局部窗口注意力           │
              └─────────────┬─────────────┘
                            │ 64维特征
              ┌─────────────▼─────────────┐
              │      CAD Decoder          │
              │      （解码预测头）          │
              │                           │
              │  输入：骨干特征 + 查询向量    │
              │  输出：实例掩码 + 类别预测    │
              │                           │
              │  核心技术：                 │
              │  · 可学习查询向量（DETR思路）  │
              │  · 6层交叉注意力解码         │
              │  · 迭代预测（每层都输出结果） │
              └─────────────┬─────────────┘
                            │
              ┌─────────────▼─────────────┐
              │         后处理             │
              │  语义预测 + 实例预测 → 全景  │
              └─────────────┬─────────────┘
                            │
              每条线段 → (类别, 实例ID)
```

## 1.8 论文与学术定位

- **论文标题**：Point or Line? Using Line-based Representation for Panoptic Symbol Spotting in CAD Drawings
- **arXiv**：arXiv:2505.23395
- **发表会议**：NeurIPS 2025（神经信息处理系统大会，顶级 AI 会议）
- **项目主页**：`http://home.weskwong.tech/releases/vecformer/`

**论文核心贡献**：
1. 证明了"线段表示"优于"点表示"用于 CAD 图分析
2. 设计了适配向量图形的骨干网络（基于 PointTransformerV3 改编）
3. 提出了迭代预测的 CAD 解码器
